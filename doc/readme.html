Building (for desktop platforms)

Requirements

  • A compiler:
    • GCC 4.5 (for Unix) OR
    • MinGW 4.5 with MSYS (for Windows) OR
    • Microsoft Visual C++ 2008 (for Windows)
    • It may be possible to use other compilers but at the moment this is not supported. GCC/MinGW are free and open source.
  • Python 2.6 or 2.7
  • 7-Zip 9.20
  • Qt 4.7
  • Qt Creator 2.1
  • OpenSSL 1.0.0 binaries for Windows from Shining Light Productions

Always download the latest Silverlock source code from http://www.petroules.com/products/silverlock. Versions downloaded from our Git repository may be untested at the time.

Instructions

Setting up 3rd party components

Setting up the 3rd party components (such as Qt Solutions and Botan) is relatively easy to do as scripts are provided. Navigate to the 3rdparty directory and run configure.bat for Windows or configure.sh for Unix.

The next step is to compile Botan. The process varies slightly by operating system:

Windows with MinGW and MSYS Windows with Microsoft Visual C++ Unix

Open the MinGW MSYS shell and navigate to the 3rdparty/temp/botan directory. In MSYS, /C/ is equivalent to C:/.

Once you are in the Botan directory, run botan.sh or botan.sh --debug to compile in debug mode.

Open the Microsoft Visual C++ 2008 command prompt and navigate to the 3rdparty/temp/botan-msvc directory.

Once you are in the Botan directory, run botan.bat or botan.bat --debug to compile in debug mode.

Open a terminal and navigate to the 3rdparty/temp/botan directory.

Once you are in the Botan directory, run botan.sh or botan.sh --debug to compile in debug mode.

Compiling Silverlock

The next step is to compile Silverlock itself. Navigate to the silverlock directory and open the silverlock.pro file in Qt Creator. Set up a shadow build for MinGW, MSVC or GCC when prompted, select either the debug or release configurations, and then click build.

Building distribution packages

To build the installers, run deploy/win32/deploy.bat for Windows and deploy/macx/deploy.sh for Mac OS X. Installers for Linux are not currently available.
